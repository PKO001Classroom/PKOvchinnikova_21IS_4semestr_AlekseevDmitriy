# Критерии приемки проекта "Учебный журнал с учетом ФГОС СПО"

## Общие критерии
- [ ] Приложение успешно запускается локально командой `python main.py`.
- [ ] Приложение успешно собирается и запускается в Docker-контейнере. (Опционально)
- [ ] Все автоматические тесты (pytest) проходят успешно.
- [ ] README.md содержит актуальные инструкции по установке, запуску и описанию структуры.

## Функциональные критерии
### Авторизация и роли
- [ ] Возможен вход под учетной записью преподавателя (teacher1/123456).
- [ ] Возможен вход под учетной записью студента (student1/123456).
- [ ] После входа открывается интерфейс, соответствующий роли.

### Функционал преподавателя
- [ ] Преподаватель видит список студентов в выпадающем меню.
- [ ] Преподаватель может выбрать предмет, компетенцию ФГОС и индикаторы.
- [ ] При выборе индикаторов автоматически рассчитывается и отображается оценка.
- [ ] Преподаватель может добавить комментарий длиной не менее 100 символов.
- [ ] Оценка с комментарием и выбранными индикаторами сохраняется в БД.
- [ ] В журнале преподавателя отображаются все выставленные им оценки.

### Функционал студента
- [ ] Студент видит таблицу со своими оценками по всем предметам.
- [ ] Для каждой оценки отображается предмет, компетенция, оценка, процент, комментарий и дата.
- [ ] Оценки в таблице выделены цветом согласно их значению (2,3,4,5).
- [ ] Студент может просмотреть детальную информацию по выбранной оценке (индикаторы, развернутый комментарий).
- [ ] Отображается средний балл студента.

### Логика ФГОС
- [ ] Оценка выставляется не напрямую, а через выбор освоенных индикаторов.
- [ ] Система автоматически пересчитывает процент освоения и оценку по шкале ФГОС (2-5) на основе выбранных индикаторов.
- [ ] Для компетенции с 8 индикаторами действует правило: 6-8 инд. -> "5", 5 инд. -> "4", 4 инд. -> "3", 0-3 инд. -> "2".

## Нефункциональные критерии
- [ ] В репозитории нет служебных файлов (__pycache__, .pyc)
- [ ] Документация в папке `documentation` полная и непротиворечива.
