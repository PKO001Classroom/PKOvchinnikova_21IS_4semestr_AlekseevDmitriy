# Техническое задание на разработку приложения для управления учебным процессом (PyQt5 версия)

## 1. Введение

### 1.1. Назначение документа
Настоящее техническое задание описывает требования к desktop-приложению для управления учебным процессом, предназначенному для взаимодействия преподавателей и студентов колледжей.

### 1.2. Цель проекта
Разработка desktop-приложения на PyQt5 для ведения электронного журнала оценок с разграничением прав доступа между преподавателями и студентами, с учетом требований ФГОС СПО.

## 2. Функциональные требования

### 2.1. Общий функционал
- **Авторизация пользователей** по логину/паролю с определением роли
- **Верификация данных** профиля при входе
- **Уведомление** о несовпадении данных с предложением повторного ввода
- **Ролевой доступ** к функциям системы

### 2.2. Функционал для студента
- **Просмотр оценок** по всем предметам в табличном виде
- **Просмотр комментариев** от преподавателей к каждой оценке
- **Просмотр списка предметов** и информации о них
- **Просмотр среднего балла** по предметам и в целом
- **Цветовая индикация** оценок (зеленый для 4-5, желтый для 3, красный для 2)
- **Отсутствие возможности** выставлять или изменять оценки

### 2.3. Функционал для преподавателя
- **Просмотр списка студентов** в системе
- **Просмотр оценок** студентов по всем предметам
- **Просмотр предметов**, доступных для преподавания
- **Просмотр комментариев** к оценкам
- **Выставление оценок** по системе ФГОС (2, 3, 4, 5)
- **Написание комментариев** к оценкам (обязательно не менее 10 символов)
- **Обновление данных** в реальном времени

### 2.4. Хранение данных
- **Оценки** с привязкой к студенту, преподавателю, предмету, дате
- **Комментарии** к оценкам с рекомендациями по компетенциям ФГОС
- **Предметы** учебного плана с кодами по ФГОС
- **Профили пользователей** (преподаватели и студенты) с информацией о специальности и группе
- **Автоматический расчет** средних баллов

## 3. Бизнес-логика

### 3.1. Процесс авторизации
1. Пользователь вводит логин и пароль
2. Система проверяет соответствие с данными в базе
3. При несовпадении - показ сообщения об ошибке
4. После успешной аутентификации определяется роль
5. Открывается соответствующий интерфейс

### 3.2. Работа с оценками
- Оценки выставляются только преподавателями
- Система оценок соответствует ФГОС (2-5 баллов)
- К каждой оценке прикрепляется комментарий (минимум 10 символов)
- Рекомендуется указывать код компетенции (ПК/ОПК) в комментарии
- Автоматический расчет средних баллов при добавлении оценок
- Цветовая индикация оценок для наглядности

### 3.3. Выход из системы
- Корректное завершение сессии
- Сохранение всех внесенных изменений
- Закрытие подключения к базе данных

## 4. Нефункциональные требования

### 4.1. Производительность
- Время отклика на основные операции не более 1 секунды
- Поддержка одновременной работы на одном компьютере
- Быстрая загрузка интерфейса

### 4.2. Безопасность
- Локальное хранение данных (SQLite)
- Простая аутентификация (для демо-версии)
- Ролевая модель доступа
- Защита от некорректного ввода данных

### 4.3. Надежность
- Стабильная работа без зависаний
- Автоматическое создание резервных копий (опционально)
- Восстановление после сбоя

## 5. Интерфейс пользователя

### 5.1. Общие требования
- Интуитивно понятный интерфейс
- Современный дизайн в стиле Material Design
- Поддержка русского языка
- Адаптация под разные разрешения экрана

### 5.2. Разделы интерфейса
- **Окно авторизации** - поля ввода, кнопки, подсказки
- **Окно студента** - таблица оценок, фильтры, статистика
- **Окно преподавателя** - журнал оценок, форма добавления, управление

### 5.3. Визуальные элементы
- Цветовая схема: синий/зеленый для позитивных элементов, красный для ошибок
- Иконки для основных действий
- Таблицы с чередованием строк
- Информативные сообщения об ошибках и успехах

## 6. Технические требования

### 6.1. База данных
- SQLite 3.x
- Структура таблиц: пользователи, предметы, оценки
- Локальное хранение в папке data/

### 6.2. Бэкенд
- Язык программирования: Python 3.11+
- Библиотека для GUI: PyQt5 5.15.9
- Для работы с БД: sqlite3 (встроенная)

### 6.3. Фронтенд
- PyQt5 для интерфейса
- QTableWidget для таблиц
- QComboBox для выпадающих списков
- QMessageBox для уведомлений

## 7. Этапы разработки

1. **Проектирование БД и архитектуры** - 1 неделя
2. **Разработка ядра приложения** (база данных, модели) - 1 неделя
3. **Разработка интерфейса** (окна, формы) - 2 недели
4. **Интеграция компонентов** и бизнес-логики - 1 неделя
5. **Тестирование и отладка** - 1 неделя
6. **Упаковка и документация** - 1 неделя

## 8. Критерии приемки

- Реализована полная функциональность согласно требованиям
- Приложение запускается без ошибок
- Все основные сценарии использования работают корректно
- Интерфейс интуитивно понятен
- Данные сохраняются корректно
- Ролевой доступ работает правильно
